32.1.1 IPA Protocol Objects ¶

The commands sent to and results received from agent may contain some complex data types called objects.

The in-process agent is running on the same machine with ROCgdb or GDBserver, so it doesn’t have to handle as much differences between two ends as remote protocol (see ROCgdb Remote Serial Protocol) tries to handle. However, there are still some differences of two ends in two processes:

  1. word size. On some 64-bit machines, ROCgdb or GDBserver can be compiled as a 64-bit executable, while in-process agent is a 32-bit one.
  2. ABI. Some machines may have multiple types of ABI, ROCgdb or GDBserver is compiled with one, and in-process agent is compiled with the other one.

Here are the IPA Protocol Objects:

  1. agent expression object. It represents an agent expression (see The GDB Agent Expression Mechanism).
  2. tracepoint action object. It represents a tracepoint action (see Tracepoint Action Lists) to collect registers, memory, static trace data and to evaluate expression.
  3. tracepoint object. It represents a tracepoint (see Tracepoints).
